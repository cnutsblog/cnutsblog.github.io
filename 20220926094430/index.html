SchildiChat使用指南

发表于 2022-09-26 更新于 2022-09-28

　　iOS用户和想要粗略了解Matrix的同学请阅读Matrix客户端使用指南。

　　SchildiChat是知名Matrix客户端Element的衍生版本。虽然图标和UI比Element丑陋，却拥有比Element更加简便的登录方式和更丰富的功能。SchildiChat和Element一样有多个平台的客户端以及网页端(SchildiChat暂时没有iOS端 😢)。这篇文章将教会你从注册到使用快速上手SchildiChat。

准备工作

Homeserver

(能够正常访问Matrix.org且不考虑访问速度的同学可以跳过该步骤)

　　众所周知，Matrix是一个协议，服务的供应商取决于注册账号时填写的homeserver。一般情况下大家会选择matrix.org作为homeserver。由于部分地区并没有能力直接访问matrix.org，且Matrix组织也呼吁不要把账号都放在同一个供应商里，因此你需要找到一个便于自己访问的homeserver并在上面注册账号。

　　joinmatrix.org和asra.gr’s list列出了开放注册的homeserver。建议先挑选一个地理位置离自己较近的，然后在ITDOG输入homeserver域名并观察居住地服务器访问该域名的延迟(<=200ms的线路比较顺畅)。如果捉摸不定，这里也有一个清单可供参考。

  • 居住地在亚太、美洲: 🇯🇵comm.cx 🇺🇸mozilla.org 🇺🇸community.rs

  • 居住地在中西亚、欧洲、非洲: 🇷🇺sibnsk.net 🇩🇪envs.net 🇩🇪yatrix.org

　　如果能够自行搭建homeserver是最好的，使用质量和数据安全都能够得到保证。

Matrix账号

(能够正常访问Google的同学可以跳过该步骤，直接使用Schildi注册账号)

　　部分homeserver在注册时会弹出reCaptha验证，由于Element/Schildi转发reCaptha页面会出错，导致不能正常访问Google的用户也无法在Schildi通过reCaptha验证。

　　FluffyChat会通过浏览器打开homeserver提供的reCaptha，不会出现错误。因此我们可以通过FluffyChat的网页端注册一个账号，再通过SchildiChat登录。如果reCaptha仍无法加载，再通过Schildi注册也无妨。

　　打开FluffyChat的网页端(或通过官网下载客户端)，将homeserver一栏(默认是matrix.org)替换成你选好的homeserver的域名。点击”Connect”，填写用户名注册。

　　注册成功后你会得到一个MXID，记格式为@username:homeserver.domain，username是注册时填写的用户名，homeserver.domain是你的账号所在homeserver的域名。记牢MXID和密码就可以无阻碍地登录Schildi了🎉。

F-Droid应用管理器

(没有打算在安卓/鸿蒙设备使用SchildiChat的同学请忽略该步骤)

　　F-Droid是安卓平台的一款功能强大的开源软件管理器。可以通过添加各种软件仓库来实现快速下载和更新各类软件。F-Droid将用于更新Schildi和消息推送器ntfy。

　　从官网下载apk安装F-Droid，更改官方仓库的镜像地址以加快更新速度(例如TUNA)并关闭其他的下载源。之后你就可以定期更新SchildiChat了。

　　看不懂也没关系，这里有更详细的教程。

UnifiedPush(ntfy)

(没有打算在安卓/鸿蒙设备使用SchildiChat的同学请忽略该步骤)

　　由于SchildiChat/Element自带的后台同步功能会显著加快移动设备的电量消耗。因此我们需要通过UnifiedPush协议代理SchildiChat的消息推送。

　　官网推荐使用ntfy作为UnifiedPush客户端。打开F-Droid，搜索ntfy并安装。打开ntfy一次然后关闭，应用设置允许ntfy发送通知、开机自启动并关闭电池优化(如果找不到电池优化选项，在ntfy拥有订阅之后再打开ntfy，会有相关提示，进入页面关闭即可)。

　　打开设置，将链接协议改为Websockets。

上手

　　拿出在FluffyChat事先注册好的MXID，格式记为@username:domain.name。

SchildiChat有两种UI，分为安卓端和桌面/网页端。

安卓

安装

　　打开事先准备好的F-Droid，搜索SchildiChat，下载并安装。

登录

　　点击“开始吧”之后会有三种登录方式供选择，点击中间的“自定义服务器”(如果决定使用matrix.org请点击最下方按钮)。填入MXID中domain.name部分。

　　一般情况下会出现“注册”和“登录”按钮。点击“登录”，填入MXID中username部分，输入密码，就可以登录了。只有一个“单点登录”按钮时，点击并按照指示登录即可。

　　FluffyChat注册失败，或使用matrix.org作为homeserver的同学可以点击注册按钮注册账号。对于非matrix.org的使用者，如果在Schildi同样遇到reCaptha加载失败的问题，考虑以下备选方案: 1.通过网络代理等方式更改ip至其他地区，通过reCaptha之后再切换回正常的ip，只要homeserver能够直接访问，今后都可以正常登录和使用。2.将homeserver改为mozilla.org登录。

配置

　　登录成功时会弹窗选择后台同步方式，选择“ntfy”即可。如果没有弹窗，打开ntfy，再将Schildi的通知方式配置为ntfy(首页左上角三条横杠->设置->通知->通知方式)。打开ntfy，检查订阅条目是否增加，如果弹出关闭电池优化的请求，点击同意并在弹出的设置页面找到ntfy条目关闭电池优化。

　　点击首页左上角三条横杠打开设置，参考下列图片和建议，调整自己的客户端。

　　接下来就可以愉快地加入聊天室了🎉。如果遇到不会使用的情况，参照下图。

桌面/网页

安装

　　打开下载页面，按照提示点击自己平台对应的下载链接。

　　由于Element/SchildiChat在登录时默认访问matrix.org加载必要的资源，无法访问matrix.org的客户端会出现“Invalid homeserver discovery response”等错误，导致启动失败。Element/SchildiChat会优先读取用户配置目录下的配置文件，在用户配置目录添加config.json并修改默认homeserver以避免访问matrix.org。

　　按照Element用户配置指南，打开自己的操作系统所对应的用户配置路径，注意$NAME对应的值在这里是SchildiChat，举例: 我用Windows系统，那么我的用户配置路径默认是%APPDATA%\SchildiChat\config.json(如果找不到路径，可以自行建立文件夹)。

　　进入config.json所在的文件夹后，新建config.json(保险起见可以复制同文件夹的其他.json文件并更名)并用记事本打开，将模板复制粘贴到记事本，找到"default_server_name": "matrix.org"一栏，将matrix.org改为mozilla.org(也可以是aria.im等能够连通的homeserver)，保存关闭。

　　嫌麻烦的可以直接复制以下内容到config.json中。

　　之后再启动SchildiChat就不会有错误提示了。动手能力差或是听不懂这段教程的也可以通过特殊手段临时实现访问matrix.org，待使用自己的homeserver登录后再恢复正常方式，只要成功登录一次，之后都可以直接使用😜。

　　使用官方网页端必须拥有能够访问matrix.org的能力。

登录

　　进入主页面后，点击左边的“Sign In”，点击“Edit”将homeserver改为你的MXID中homeserver.domain的部分。填写username、密码即可登录。如果homeserver要求单点登录，点击并按照要求操作即可。

　　FluffyChat注册失败，或使用matrix.org作为homeserver的同学可以点击“Create Account”按钮注册账号。对于非matrix.org的使用者，如果在Schildi同样遇到reCaptha加载失败的问题，考虑以下备选方案: 1.通过网络代理等方式更改ip至其他地区，通过reCaptha之后再切换回正常的ip，只要homeserver能够直接访问，今后都可以正常登录和使用。2.将homeserver改为mozilla.org登录。

配置

　　首次登录会弹出提示，是否加入更新推送频道，点击“Show Preview”，然后加入即可。

　　点击首页左下角打开设置，参考下列图片和建议，调整自己的客户端。

　　接下来就可以愉快地加入聊天室了🎉。

使用

通过外部链接加入聊天室

外部链接的格式是https://matrix.to/#/#room(@username/!id):homeserver.name，#/@/!及之后的内容即为聊天室地址/MXID/ROOMID。打开外部链接时会弹出选择客户端的提示，选择Element并继续即可。

推荐的聊天室

  • #fujianlongface:comm.cx
